🧭 시작 전에 — 데이터 분석이 처음이라면?

데이터 분석은 "쌓여있는 숫자에서 의미 있는 패턴이나 답을 찾아내는 과정"입니다. 이번 강의의 목표는 반도체 공정 센서값(온도·압력·전력 등)을 보고 "이 제품은 합격일까, 불합격일까?"를 자동으로 예측하는 모델을 만드는 것입니다.

3단계 · 결측값(Missing Value) 처리
비어있는 데이터를 채우거나 제거합니다
🕳️
결측값(NaN)이란? 센서가 잠깐 꺼지거나, 입력이 누락되면 그 칸은 "비어 있음" 상태가 됩니다. pandas에서는 이것을 NaN(Not a Number)으로 표시합니다. 비어 있는 값이 그대로 있으면 평균·표준편차 같은 계산이 깨지거나, 머신러닝 모델이 학습을 거부합니다. 그래서 분석 전에 반드시 채우거나 지워야 합니다.
처리 방법코드언제 사용?

print(df.isnull().sum())

🔍 히트맵, 이렇게 읽으세요 — 단계별 가이드

① 행과 열에 같은 변수들이 같은 순서로 놓입니다

예를 들어 12개 변수가 있으면 12행 × 12열의 정사각형 표가 만들어집니다. (i, j) 칸의 숫자는 i번째 변수와 j번째 변수의 상관계수를 뜻하고, 보통 −1.0 ~ +1.0 사이의 값입니다.

② 색깔로 관계의 방향과 세기를 한눈에 봅니다

색상관계수 범위의미
진한 초록+0.7 ~ +1.0한 값이 커지면 다른 값도 함께 커진다 (강한 양의 상관)
연한 초록+0.3 ~ +0.7같은 방향으로 어느 정도 움직인다 (중간 양의 상관)
연한 노랑·하양−0.3 ~ +0.3거의 관계 없음 (무관)
연한 빨강−0.7 ~ −0.3한 값이 커지면 다른 값은 줄어든다 (중간 음의 상관)
진한 빨강−1.0 ~ −0.7한 값이 커지면 다른 값은 크게 줄어든다 (강한 음의 상관)

③ 대각선은 항상 1.0이라 의미가 없습니다

(i, i) 칸은 변수 자기 자신과의 상관이라 무조건 1.0입니다. 분석할 때는 대각선 위/아래만 보면 충분합니다 (위·아래는 같은 값이라 한쪽만 봐도 됩니다).

④ "합격여부" 줄(또는 열) 한 줄에 집중하세요

이번 분석의 목표는 합격 여부를 예측하는 것이므로, 다른 변수가 합격여부와 얼마나 관련 있는지가 핵심입니다. 절댓값이 큰 변수(색이 진한 칸)일수록 예측에 큰 영향을 줍니다.

📖
예시로 손에 익히기 만약 히트맵의 "온도_섭씨 × 합격여부" 칸 값이 −0.42라면?
• 부호가 음(−) → 온도가 높을수록 불합격이 늘어나는 경향이 있다는 뜻
• 절댓값 0.42 → 중간 정도의 영향력이 있다는 뜻
⇒ 공정에서 온도를 잘 관리하는 것이 합격률에 중요할 가능성이 큽니다.
⚠️
상관관계 ≠ 인과관계 "두 값이 같이 움직인다"는 사실이 곧 "한쪽이 다른 쪽의 원인이다"라는 뜻은 아닙니다. 숨겨진 다른 변수(계절, 장비 노후, 작업자 차이 등) 때문에 동시에 변하는 경우도 많아요. 상관관계는 "의심 가는 변수"를 찾는 출발점일 뿐, 결론은 실험·검증으로 확인해야 합니다.

Python – 데이터 분리 & 스케일링 
X = df[numeric_cols].copy()
y = (df['합격여부'] == 1).astype(int)  # 1=합격, 0=불합격

# 학습/테스트 분리
X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    test_size=0.2,       # 20% 테스트용
    random_state=42,    # 재현성 시드
    stratify=y          # 합격/불합격 비율 유지
)

# 스케일링 (단위 통일)
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)  # 학습 + 변환
X_test_scaled  = scaler.transform(X_test)        # 변환만!
⚠️
테스트 데이터에 fit()을 하면 안 됩니다! 테스트 데이터로 fit()하면 미래 데이터의 정보가 모델에 유출됩니다 (데이터 누출, Data Leakage). 반드시 학습 데이터로만 fit하고, 테스트 데이터는 transform만 해야 합니다.

🔧 위 코드에 등장한 함수 한 줄씩 풀기

① df[numeric_cols].copy()

원본 DataFrame이 변하지 않도록 복사본을 만듭니다. 복사하지 않고 슬라이싱한 결과를 수정하면 원본도 같이 바뀌어 SettingWithCopyWarning이 뜨고, 디버깅이 까다로워집니다.

② (df['합격여부'] == 1).astype(int)

원본은 합격=1 / 불합격=−1로 저장돼 있지만, 머신러닝에서는 0과 1로 다루는 것이 표준입니다.
== 1 → 각 행이 True/False가 되고, .astype(int)로 1/0 정수로 바꿉니다.

③ train_test_split(X, y, ...)

전체 데이터를 학습용·테스트용으로 자동 분리합니다. 각 인자의 의미는 다음과 같습니다.

  • test_size=0.2 — 전체의 20% (40행)를 테스트용으로 떼어 둡니다.
  • random_state=42 — 무작위로 나누되 매번 같은 결과가 나오도록 시드를 고정합니다. (수업·실험 재현에 필수)
  • stratify=y — 합격/불합격 비율을 학습·테스트 양쪽에 똑같이 유지합니다. 이 옵션이 없으면 우연히 테스트에 불합격이 0개일 수도 있어요.

④ StandardScaler() — 표준화

각 컬럼의 값에서 평균을 빼고 표준편차로 나눠 평균 0, 표준편차 1로 맞춥니다.

왜 필요한가? 온도(290~310)와 압력(1000~1500)처럼 숫자의 크기 차이가 크면, 큰 단위의 변수가 모델에 과도하게 영향을 줍니다. 모두 같은 척도로 맞춰야 공정한 비교가 가능합니다. (특히 로지스틱 회귀, KNN, SVM 같은 모델은 스케일에 민감합니다.)

⑤ fit_transform() vs transform()

• fit(): "이 데이터의 평균과 표준편차를 계산해서 기억해 둬!"
• transform(): "기억해 둔 값으로 실제로 변환해!"
• fit_transform(): 위 두 가지를 한 번에 수행.
학습 데이터에는 fit_transform() (계산 + 변환), 테스트 데이터에는 transform()만 적용합니다. 그래야 테스트 데이터의 통계 정보가 학습에 새지 않습니다 (데이터 누출 방지).

이번 6단계의 흐름은 단순합니다:

  1. 이미 답(합격/불합격)을 아는 학습 데이터 160개를 모델에 보여준다.
  2. 모델은 "어떤 센서값일 때 합격이지?"를 스스로 학습한다.
  3. 학습이 끝난 모델에 새 데이터(테스트 40개)의 입력만 주면, 합격/불합격을 예측한다.

📐 두 모델이 답을 찾는 방식

📈 로지스틱 회귀 (Logistic Regression)

데이터 점들을 하나의 직선(또는 평면)으로 나눕니다. "이 점은 직선의 어느 쪽에 더 가깝나?"로 합격·불합격을 판단해요.
→ 결정 경계가 단순할 때 빠르고, 결과 해석이 쉽습니다.

🌲 랜덤 포레스트 (Random Forest)

수많은 "질문 트리(decision tree)"를 만들어 다수결 투표로 답을 결정합니다. 한 트리는 다음과 같이 질문해 갑니다.

Q1. 온도 > 305°C ? → Yes
   Q2. 두께 > 103 nm ? → Yes → 🔴 불합격
   Q2. 두께 > 103 nm ? → No  → 🟢 합격

이런 트리 100개의 투표 결과로 최종 답을 결정합니다. 복잡한 패턴도 잡아내고 이상치에 강한 것이 장점이에요.

⚙️ 모델 옵션(하이퍼파라미터) 풀이

옵션의미너무 작으면 / 너무 크면
n_estimators=100만들 결정 트리의 개수적으면 결과가 불안정, 많으면 학습 속도가 느려짐 (보통 100~500)
max_depth=5한 트리의 최대 깊이깊을수록 복잡한 패턴까지 학습 / 너무 깊으면 학습 데이터만 외움 (과적합)
random_state=42난수 시드같은 값이면 매번 같은 결과 → 재현 가능
max_iter=1000로지스틱 회귀의 최대 반복 학습 횟수적으면 "수렴하지 못함" 경고가 발생
🪄
fit()과 predict() 한 줄 요약 model.fit(X_train, y_train) → "이 데이터로 공부해!" (학습)
model.predict(X_test) → "공부한 걸로 정답 맞춰봐!" (예측)
📈
로지스틱 회귀
통계 기반 선형 분류 모델. 빠르고 해석이 쉬움. 선형 경계일 때 적합.
🌲
랜덤 포레스트
수백 개의 결정 트리가 투표. 비선형 패턴 학습 가능. 이상치에 강함.
Python – 모델 학습 
# 로지스틱 회귀
lr_model = LogisticRegression(random_state=42, max_iter=1000)
lr_model.fit(X_train_scaled, y_train)

# 랜덤 포레스트
rf_model = RandomForestClassifier(
    n_estimators=100,   # 결정 트리 100개
    max_depth=5,        # 각 트리 최대 깊이
    random_state=42
)
rf_model.fit(X_train_scaled, y_train)

# 예측
y_pred = rf_model.predict(X_test_scaled)
🎯
7단계 · 모델 평가
혼동행렬, ROC-AUC, 특성 중요도로 모델을 분석합니다
📚
혼동행렬을 일상에 비유하면? 학교 시험에서 선생님이 학생을 "합격/불합격"으로 채점한다고 해 봅시다. 모델의 예측과 실제 정답을 짝지어 보면 네 가지 결과가 나옵니다.
• TP (True Positive) — 합격이라 예측 → 진짜 합격 ✅
• TN (True Negative) — 불합격이라 예측 → 진짜 불합격 ✅
• FP (False Positive, 거짓 경보) — 합격이라 예측했는데 실제는 불합격 ❌
• FN (False Negative, 놓침) — 불합격이라 예측했는데 실제는 합격 ❌

📌 혼동행렬(Confusion Matrix) 읽는 법

TN
불합격 → 불합격
✅ 정확
FP
불합격 → 합격
⚠️ 틀린 경보
FN
합격 → 불합격
❌ 누락
TP
합격 → 합격
✅ 정확
지표공식의미
Accuracy(TP+TN) / 전체전체 정확도
PrecisionTP / (TP+FP)불합격 예측의 정확성
RecallTP / (TP+FN)실제 불합격 탐지율
F1-Score2 × P×R / (P+R)Precision·Recall 조화
AUCROC 곡선 아래 면적1에 가까울수록 좋음
Python – 모델 평가 
# 혼동행렬
cm = confusion_matrix(y_test, y_pred)
sns.heatmap(cm, annot=True, fmt='d', cmap='Blues')

# 상세 성능 리포트
print(classification_report(y_test, y_pred,
      target_names=['불합격(0)', '합격(1)']))

# 특성 중요도
importances = pd.Series(
    rf_model.feature_importances_,
    index=numeric_cols
).sort_values(ascending=False)
print(importances)

🎯 평가 지표, 한 번 더 쉽게

  • Accuracy (정확도) — 전체 중 몇 %를 맞혔나? 불균형 데이터에서는 함정이 큼
  • Precision (정밀도) — "이건 양성!"이라고 외친 것 중 진짜 양성은 몇 %? 잘못된 알람을 줄이고 싶을 때
  • Recall (재현율) — 진짜 양성 중 몇 %를 모델이 잡아냈나? 놓치면 안 될 때 중요 (예: 불량품)
  • F1-Score — Precision과 Recall의 조화평균. 둘 다 균형 있게 높아야 1에 가까워짐
  • Support — 그 클래스에 속한 실제 데이터의 개수
💡
반도체 공정에서는 Recall이 중요합니다! 불합격 제품을 합격으로 예측하면(FN) 불량품이 출하됩니다. 이는 Precision이 낮은 것보다 훨씬 심각한 문제입니다. 따라서 이 도메인에서는 Recall을 최대화하는 방향으로 모델을 조정해야 합니다.

겉보기 97% 정확도에 숨은 함정 Accuracy 0.97은 매우 높아 보이지만, 불합격 4개 중 1개를 놓쳤습니다 (Recall = 0.75). 반도체 공정에서는 불량품을 합격으로 판정하면 출하 사고로 직결되므로, 이 모델은 아직 충분히 안전하지 않습니다. 개선 방향:
① 예측 임계값을 낮춰 불합격을 더 적극적으로 잡기
② class_weight='balanced' 옵션으로 불합격에 가중치 부여
③ SMOTE 등으로 불합격 샘플을 오버샘플링해 학습
